# 🛡️ End-to-End Anti-Money Laundering (AML) Detection Pipeline

## 📌 Project Overview

This project builds a complete **machine learning pipeline for detecting fraudulent financial transactions (Anti-Money Laundering detection)**.

The system is designed to:

* Detect suspicious transactions in highly imbalanced financial data
* Reduce false negatives (missed fraud cases)
* Adapt to evolving fraud patterns using continuous learning
* Combine unsupervised + supervised learning techniques

---

## 🎯 Business Problem

Financial institutions process millions of transactions daily. Traditional rule-based AML systems:

* Generate a large number of false alerts
* Miss hidden fraud patterns
* Do not adapt to new fraud behaviors

### ⚠️ Objective:

Build a machine learning system that:

* Accurately detects fraudulent transactions
* Reduces false positives
* Maximizes fraud detection (Recall priority)
* Continuously updates over time

---

## 📊 Dataset

We use the **PaySim / IBM AML synthetic transaction dataset**, which contains:

* Transaction types (TRANSFER, CASH_OUT, PAYMENT, etc.)
* Account information
